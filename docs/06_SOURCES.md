# Thinking in Wholes — Sources

> The working bibliography. Every entry is keyed to the chapter(s) where it appears, with **verification status** and **role**.
>
> **Verification status** (mandatory for every entry):
> - ✓ **VERIFIED** — confirmed via authoritative source (publisher, library catalog, archive). Safe to cite in drafts.
> - ⏳ **PARTIAL** — work confirmed to exist, but some bibliographic detail (edition, page, publisher city) needs confirmation. Note what's missing.
> - ✗ **UNVERIFIED** — claimed by AI memory but not yet checked. **DO NOT CITE in drafts until verified.**
>
> **Role**:
> - **[P]** — primary source (text we engage with directly)
> - **[S]** — secondary source (background, supporting evidence)
>
> Format follows Oxford referencing (see `02_STYLE_GUIDE.md`).
>
> **Initial state:** all entries seeded as ✗ UNVERIFIED from AI training memory. Verification batches (per `04_PROGRESS.md`) move entries to ✓ VERIFIED before they are cited in any draft.

## Verification protocol (process notes)

When running a verification batch, for each ✗ UNVERIFIED entry:

1. **Web-search** the work using the most distinctive identifier available (full title plus author last name).
2. Cross-check with **at least one authoritative source**: the publisher's catalog, a major academic library catalog (e.g., Library of Congress, British Library, WorldCat), Google Scholar, or the work's official website.
3. Confirm the following minimum bibliographic data:
   - Author full name (with correct initials and order)
   - Exact title (including subtitle, capitalization)
   - Publisher
   - City of publication
   - Year of publication (and edition if multiple exist)
4. If verifying for direct quotation: also confirm the quotation text and the page number against the actual source. (For older works, public-domain texts are usually accessible online; for in-copyright works, this may require partial verification with a strong note that page numbers must be checked against a physical or digital copy before final publication.)
5. Update the entry's status marker:
   - All five fields confirmed → ✓ **VERIFIED**
   - Some fields confirmed but gaps remain → ⏳ **PARTIAL** (note what's missing)
   - Cannot confirm the work exists → leave as ✗ **UNVERIFIED**, and add a note: *Could not verify; recommend cutting or replacing.*
6. Log the verification work done in `04_PROGRESS.md` under the relevant batch.

**Special note on AI memory:** Particular caution is required for: (a) less-famous primary sources, (b) specific edition/year details, (c) journal article volume/issue numbers, (d) page numbers in any direct quotation, (e) Edgar A. Singer Jr.'s bibliography, and (f) any source claimed from before 1950. AI memory is least reliable in these areas.
